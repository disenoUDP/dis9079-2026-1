## Investigación sobre APIs

Las API (interfaces de programación de aplicaciones) son mecanismos que permiten que diferentes aplicaciones o componentes de software se comuniquen entre sí mediante solicitudes y respuestas. Funcionan como un puente entre un cliente, que realiza una petición, y un servidor, que entrega la información o servicio solicitado. Gracias a las API, los desarrolladores pueden integrar datos, servicios y funcionalidades de otras aplicaciones sin necesidad de crearlos desde cero, lo que simplifica y acelera el desarrollo de software.

Para que una API sea fácil de utilizar, es fundamental contar con una documentación clara que explique sus funciones, protocolos, métodos de autenticación y la forma correcta de estructurar las solicitudes y respuestas.

Un ejemplo cotidiano es el pago mediante plataformas externas en sitios web. Cuando un usuario selecciona un método de pago, el sitio envía una solicitud a través de una API al servicio de pago correspondiente, este procesa la información y devuelve una respuesta con el resultado de la transacción. Todo este proceso ocurre de forma transparente para el usuario.

Existen distintos tipos de API, como SOAP, RPC y REST. Entre ellas, las API REST son las más utilizadas actualmente, ya que permiten intercambiar datos mediante HTTP utilizando métodos como GET, PUT y DELETE. Además, son "sin estado", lo que significa que el servidor no almacena información del cliente entre solicitudes.


#### Tipos de API

API privadas: Internas de una empresa y solo se utilizan para conectar sistemas y datos dentro de la empresa.

API públicas: Están abiertas al público y pueden cualquier persona puede utilizarlas. Puede haber o no alguna autorización y coste asociado a este tipo de API.

API de socios: Solo pueden acceder a ellas los desarrolladores externos autorizados para ayudar a las asociaciones entre empresas.

API compuestas: Estas combinan dos o más API diferentes para abordar requisitos o comportamientos complejos del sistema.
